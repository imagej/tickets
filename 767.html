The issue is that when IJ1 saved the file, it still embedded the number of channels, frames and slices into the TIFF metadata. I am not sure how IJ1 knows to display the file as a regular stack rather than hyperstack, since there is no obvious metadata field declaring to do so.

However, I am not very worried about this problem. Too much information (e.g., discrete dimensional axes) is much less of an issue than too little (e.g., axes being mushed together). Users can always combine the axes together afterward using a restructuring plugin if desired.
