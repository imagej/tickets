Description

It would excellent to have jar2lib running as a Maven plugin, so that you can invoke it on any Maven project and create a full set of C++ bindings. It could define standard Maven locations below the src directory for the configuration files (conflicts list and header template). A Maven-based approach would also mitigate classpath issues since every Maven project fully declares its dependencies, and they all could be included in AutoProxy's classpath.
