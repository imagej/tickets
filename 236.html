Description (last modified by rlentz)

Several methods exists for integrating software services on multiple platforms. A goal of the ImageJ2 effort is to enable standardized integration across multiple software efforts. Web services (binary or other) may be useful for integrating new technologies.

A demo (1) will be prepared for binary web service (hessian 4.0.7) for a plugin.
A demo (2) will be prepared for format conversion using published web service (WSDL) between two bioformats files.

These web service based plugins should be discoverable (from within ImageJ). The final demo (3) will show how the two above web services can be consumed to operate within existing core IJ code. The hessian plugin will be made discoverable. The landing page imagejdev.org/webservices will outline this effort and these findings.
