comment:2 Changed 7 years ago by curtis

In order to  create a release version of a Maven-based project, all dependent projects and plugins must be locked in to release versions as well (i.e., no SNAPSHOT dependencies). Since this was the first official release of ImageJ2, we consequently had to release quite a few upstream projects:

Project Version Artifact
 cpptasks-parallel v20120403 cpptasks-parallel-1.0-beta-5-parallel-1-20120403.jar
 maven-nar-plugin v20120403 maven-nar-plugin-2.1-20120403.jar
 imagej v2.0.0-beta1 ij-app-2.0.0-beta1-application.zip
 imglib v2.0.0-beta1 imglib2-2.0.0-beta1.jar
 scifio v4.4-imagej-2.0.0-beta1 scifio-4.4-imagej-2.0.0-beta1.jar
 jar2libv1.0.0 jar2lib-1.0.0.jar
 cppwrap-maven-plugin v1.0.0 cppwrap-maven-plugin-1.0.0.jar

Future releases will be easier, since we need not always create new release versions of all of the above projects every time.
