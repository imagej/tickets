Description (last modified by bdezonia) (diff)

Norbert Vischer  writes:

when analyzing hyperstacks, we use a fixed order of channels, e.g:

  CH1 = PhaseContrast,
  CH2 = FluorDAPI
  CH3 = FluorGFP

If the hyperstack has the wrong order, we first have to rearrange it,
and a built-in menu with the dialog:

        getString("Change Channel Order", "1-2-3");

would be much more convenient than fiddling with "Split" and "Merge"
channels, and would automatically preserve each channel's LUT,
Brightness, Labels etc.

A data rearrangement command, allowing positions along a dimensional axis to be changed, would be of general utility and solve the use case above.
