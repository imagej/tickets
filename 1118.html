IJ2 can overwite Fiji's ImageJ.cfg

Reported by: bdezonia Owned by: dscho

Description

IJ2 has a new ImageJ.cfg file format. When you run IJ2 from Fiji via "./ImageJ --ij2" IJ2 and/or the IJ2 launcher writes the new format ImageJ.cfg in the Fiji directory. Thus Fiji cannot be run unless that .cfg file is deleted first.

The move of the ImageJ launcher to ImageJ2's repository is now finished. For convenience, there are two scripts in fiji.git now, download-launchers.sh (to download the launchers as built by ImageJ2's Jenkins) and build-launcher.sh (to build a local launcher from ImageJ2's sources; a checkout of modules/imagej2 is required, of course, and it will use Maven).
