Description

The IJ1 Overlay Options dialog allows user to specify line color and fill color as arbitrary ARGB hex strings. IJ2's version in Edit::Options::Overlay has limited color support and no transparency. This is because of the limitations of the sezpoz tracked color field. Could just make color a String field but then person can enter anything in there.

Two possible approaches here:

  • make the dialog allow colors to be specified as hex strings too. make the field a text field and maybe have a callback that validates input. field should accept "brown" as well as "#ff002288".

OR

  • make ColorRGB values supported by input harvesters. The GUI version could show a color wheel and transparency field.
