Preferences should only save non-default settings

Reported by: dscho Owned by: dscho
Priority: minor Milestone: imagej2-unscheduled
Component: Core Version:
Severity: cosmetic Keywords:
Cc: Blocked By:
Blocking: #1631

Description

When settings are unchanged, chances are that the users want to have different settings when the defaults change (e.g. when the default fill color alpha for selections is changed).

The best way to allow us (upstream) to change defaults is by not bothering to save preference settings when they are unchanged, or to delete the preference settings when the user changed the value back to the default.
