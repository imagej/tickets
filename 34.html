Description

The current standalone  VisBio application possesses several useful features:

  • Visualization of N-dimensional data
  • Texture-based volume rendering
  • 3D arbitrary slicing
  • Vector-based overlays

There are ways to do most of the above already; however, VisBio uses  VisAD and it would be nice to have support for that approach within ImageJ as well.

Aim IIA of our  ImageJ proposal involves integrating VisBio into ImageJ, so these features would be a good start.

Ideally, the VisBio-related functionality could be grouped together as an add-on.
