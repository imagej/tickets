Overlays can disappear

Reported by: bdezonia Owned by: bdezonia
Priority: major Milestone: imagej2-b9-rois
Component: Legacy Compatibility Version:
Severity: serious Keywords:
Cc: Blocked By:
Blocking: #1011

Description

Open boats. Draw two overlays quickly right after loading. Wait a while and overlays go away.

I think Harmonization is happening after the overlays are drawn (since maybe it took a while for threads to terminate). And it is not seeing the overlays drawn by the user.

Change History

comment:1 Changed 7 years ago by bdezonia

  • Milestone changed from imagej-2.0-beta1 to imagej-2.0-beta2

comment:2 Changed 7 years ago by bdezonia

We may need to set a mode in IJ2 that keeps someone from adding data to an image that is currently the image of interest to an IJ1 plugin. Once IJ1 returns and synchronization is complete the lock would be lifted. Note that this could make deadlock possible if IJ1 plugin does not terminate.

On a related note right now it takes a surprisingly long time to wait for threads to terminate before synchoronization even runs. See why that is the case too.

comment:3 Changed 7 years ago by bdezonia

In a3ba7159826fbb9a60585340613d3cceb0782ef2 thread waiting time decreased. It is now harder for this bug to happen but it is still possible.
