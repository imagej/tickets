Provide ROI rendering options

Cc: G.Landini@… Blocked By:

Description

In the imagej-devel list Gabrriel Landini mentioned that the current smoothed polygon roi representation is confusing. It seems the thick outline of the JHotDraw graphics are getting in the way of seeing whether a given pixel is in or out of a ROI.

Rendering flaws are detailed in a separate ticket. But we should allow users to specify the width of the lines used to draw ROIs, A width of zero would draw in the smallest possible outline where one could more accurately see if pixels were inside or outside ROIs.
