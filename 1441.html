Description

The angle tool in ImageJ1 requires two clicks to define an angle. The current IJ2 angle tool only requires one click, but that is because it always puts the angle origin at (0, 0) in the image space. We need to fix this behavior to be consistent with ImageJ1.

Another problem is that the angle tool does not currently report status messages matching those of ImageJ1. (See ticket #827 for details.)
