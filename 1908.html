Currently, these images are opened similarly using SCIFIO as in FIJI with the Bio-Formats importer.

In both cases, they are opened as RGBA. Compositing the alpha channel makes the images look off.

IJ1 seems to be taking extra steps to process the alpha channel out, and opens this as an RGB image instead.

So we need to decide how to handle alpha channels.

Also, the color table for the 4th channel is blue in IJ2 but the Bio-Formats importer and IJ1 use white.

comment:2 Changed 6 years ago by bdezonia

Re: the 4th channel color table comment: note that IJ2's ColorTables default order is specified to be a different order than the Colors of a IJ1 CompositeImage. Thus the difference in the 4th channel between IJ2 and IJ1/Fiji. This appears to be by design.
