Syntax for accelerators is not always intuitive

Reported by: curtis Owned by: curtis
Priority: minor Milestone: imagej2-unscheduled
Component: Plugin Framework Version: 2.0.0-beta6
Severity: serious Keywords:
Cc: Blocked By:
Blocking: #1631

Description

Recently I ran into an issue where certain accelerators being specified using the @Menu annotation's accelerator property were not being parsed as expected (see 392901ad). The issue was that the shortcuts were specified as "control 4" and "control 5" but needed to be "control NUM4" and "control NUM5" to be understood by ImageJ.

The real problem was not that they weren't understood, but that ImageJ did not issue any warnings about it. To help these sorts of issues get caught sooner, we should do so. We may also want to consider expanding the allowable syntax for the accelerator strings somehow, though the specifics would require some thought.
